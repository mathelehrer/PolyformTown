# Runlevels Overview

Polyform sequences have non-negotiable exponential growth statistics, 
and concepts like constellation-limiting can only change the asymptotics 
of combinatorial explosions by reducing the exponential base. 

In order to attain tighter complexity bounds we target successive 
and towering //runlevels//, which should be thought of as boundaries 
between efficiency epochs. Runlevels are also, by definition, the 
targets of a boot sequence when turning on a computational tool. 

The computer itself has runlevel targets. In popular linux distros, 
the command line interface (CLI) is at level 3 and the Graphical 
User Interface (GUI) is at level 5. This program requires at least 
a level 3 environment for crunching numbers, and a level 5 env
for svg viewing. 

PolyformTown also has sequential runlevels. Their primary use is to 
prove important facts about the hat tiling. Facts we are interested
in can not be proven in the space of all valid polyforms, a space 
where efficiency tends to zero very quickly. Insight and cleverness 
are both necessary to make progress toward proof goals. 

In the future maybe runlevels can be used to find other interesting 
tilings. What if we can use agents to automate searches through 
tiling spaces, and analyze other polyforms in terms of their boot 
sequence statistics? Will we find other unusual shapes? 

The runlevels are: 
 
- RL0 Core functionality and constraints from vertex figures. 
- RL1 Abstraction via cyclic enclosures (i.e. hat hexagons)
- RL2 Approaching the fractal singularity (if possible)
